# MVP Scope

## MVP Goal

Build the first playable vertical slice of Sumerki.

By the end of the MVP, a player can:

- register and login
- create one kingdom
- see a basic settlement dashboard
- accumulate resources over time
- upgrade buildings
- train basic units
- send units to PvE missions
- perform simple PvP raids
- receive battle and mission reports
- choose a patron: Independent, Empire of Dusk, or Old Pact
- receive simple events and make choices

## Target Experience

The MVP should feel like a compact strategy management loop:

1. Create an account.
2. Create a kingdom.
3. Watch resources grow.
4. Spend resources on upgrades and units.
5. Send units into danger.
6. Read the outcome.
7. Make a patron or event choice that slightly changes the kingdom.

## In Scope

- Email/password authentication.
- One kingdom per user.
- Basic kingdom profile and culture.
- Resource production over time.
- Small building set.
- Basic unit training.
- PvE missions with deterministic or simple probabilistic outcomes.
- Simple PvP raids.
- Battle and mission reports.
- Patron selection.
- Simple event choices.

## Out of Scope

Do not implement these in the first MVP:

- alliances
- large province map
- seasonal world wars
- advanced diplomacy
- dark god avatar system
- trading between players
- market
- payments
- admin panel
- chat
- real-time combat
- mobile app
- complex animations
- Phaser, Pixi, or Three.js

## Non-Goals

- The MVP does not need perfect balance.
- The MVP does not need production-grade anti-cheat.
- The MVP does not need a large content library.
- The MVP does not need localization.

## Success Criteria

The MVP is successful when a new player can complete the core loop without manual database edits or developer intervention.
